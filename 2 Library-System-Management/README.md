# 📚 Library Management System (SQL Project)

## Overview

This project is a **Library Management System** built using **MySQL** to simulate real-world library operations and database management. The system is designed to manage books, members, employees, branches, and transactions such as issuing and returning books.

The project focuses on applying **SQL concepts** in a practical scenario, including joins, aggregations, stored procedures, views, and advanced queries.

## Key Features

* 📖 Manage Books Information
* 👥 Manage Members & Employees
* 🏢 Branch Management
* 🔄 Book Issue & Return Tracking
* 💰 Rental Revenue Tracking
* 📊 Branch Performance Reports
* ⭐ Active Member Identification
* ⚙️ Stored Procedures for Automation

## Database Tables

The system includes the following main tables:

* **books** — Stores book details and availability status
* **members** — Stores member information
* **employees** — Stores employee details
* **branch** — Stores branch information
* **issued_status** — Tracks issued books
* **return_status** — Tracks returned books

## SQL Concepts Used

This project demonstrates:

* Joins (INNER JOIN, LEFT JOIN)
* Aggregate Functions (COUNT, SUM)
* GROUP BY & HAVING
* Subqueries
* Stored Procedures
* CTAS (Create Table As Select)
* Data Filtering & Sorting
* Database Relationships

The goal of this project is to **practice real-world SQL queries** and understand how databases are used in **library management systems**.

This project helps strengthen:
